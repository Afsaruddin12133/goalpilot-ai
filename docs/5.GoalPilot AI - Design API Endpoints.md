## Goal and AI Flow Routes :
``` text
POST /api/goal/interview/start
```
**Purpose:** submit the goal and return the first follow-up question.

**Auth:** Require authorization: Bearer `<authToken>`

**Inputs (JSON):**

- `goal` (string, required) — the raw one-line goal submitted by the user.
- `goalDescription` (string, optional) — short extra context if provided by the user.

**Output :**

- `interviewId` (string) : server-side session id for the interview.
- `userId` (string) : authenticated user id.
- `nextQuestions` (array of question objects) : first question AI will ask. each question include : `{id, text, requiredFormat?}`
- `status` (`"in_progress"`)

``` text 
POST /api/goal/interview/answer
```
**Purpose:**  Submit one follow-up answer for an `interviewId`; returns next question or final `interview_answers` when complete.

**Auth:** Require authorization: Bearer `<authToken>`

**Inputs (JSON):**

- `interviewId` (string, required)
- `questionId` (string, required) — id from `nextQuestions`
- `answer` (string | array | boolean | number) — user’s short/structured answer
- `confirm` (boolean, optional) — if the client sends `confirm: true` At the final step, indicate user confirmed the summary

**Output:**

- If more follow-ups are required: `{ interviewId, nextQuestion: {id,text,requiredFormat?}, collectedAnswersSummary }`
- If all follow-ups are answered and **not yet confirmed**: `{ interviewId, collectedAnswersSummary, prompt: "Please confirm to generate plan" }`
- If confirmed: `{ interviewId, interview_answers }` where `interview_answers` is the machine-friendly JSON described in the contract.

``` text
POST /api/goal/plan/generate
```
**Purpose:**  Generate the validated plan JSON (Goal → Milestones → Tasks → WeeklyPlan) using a confirmed `interview_answers`.

**Auth:** Require authorization: Bearer `<authToken>`

**Input:**

- `interview_answers`  full interview JSON as per contract
- `requestId` (string, optional) — idempotency key

**Output:**
``` text
{
  "goal": { ...Goal fields... },
  "milestones": [ ... ],
  "tasks": [ ... ],
  "weeklyPlan": [ ... ],
  "scopeAdjusted": true|false
}
```

## Task Management Routes :
``` text
GET /api/tasks/today
```
**Purpose:** Return the Daily Focus View data for the authenticated user.

**Auth:** Require authorization: Bearer `<authToken>`

**Inputs:**

- `date` (YYYY-MM-DD)
- `limit` (integer, optional — defaults to 3)

**Output:**

- `date` (string)
- `topTasks` (an array of task objects up to `limit`) — each task includes: `{id, milestoneId, goalId, title, priority, estimatedTime, dueDate, completed, whyThisMatters}` where `whyThisMatters` is the AI one-line rationale from the task model.
- `todayTotalEstimatedMinutes` (integer) — sum of estimatedTime for tasks scheduled for this date
- `progressSnapshot` — small object `{ goalId, percentComplete, milestonesCompleted, timeRemainingDays }` for the primary goal represented in `topTasks`
- `refreshAt` (ISO date-time) — recommended next refresh

``` text
PATCH /api/task/:id/complete
```
**Purpose:** Mark a task complete (or toggle) and return updated task + updated goal progress.

**Auth:** Require authorization: Bearer `<authToken>`

**Inputs:**

- `completed` (boolean, required)
- `timeSpentMinutes` (integer, optional) — actual minutes spent (used for analytics)
- `autoAdvance` (boolean, optional) — if true, automatically set next task in milestone `in_progress`

**Output:**

- `task` (updated task object)
- `goalProgress` (object) — updated progress for the parent goal `{goalId, percentComplete, completedTasks, totalTasks}`
